# Public Dataset Demo

## Dataset

- Title: Online Retail
- Publisher: UCI Machine Learning Repository
- Official source: https://archive.ics.uci.edu/dataset/352/online%2Bretail
- License: Creative Commons Attribution 4.0 International (CC BY 4.0)
- Citation: Chen, D. (2015). Online Retail [Dataset]. UCI Machine Learning Repository. https://doi.org/10.24432/C5BW33.
- Download date: 2026-07-19
- Raw format: CSV sample for local demo preparation; official source provides XLSX.

The full UCI dataset contains transaction line items for a UK non-store online retailer from 2010-12-01 through 2011-12-09. FinCore uses a small processed subset so the GitHub repository stays lightweight and the demo can run on a laptop CPU.

## Files

- `data/sample/uci_online_retail_payments.json`: committed processed demo subset.
- `data/NOTICE.md`: attribution and usage notice.
- `data/raw/`: ignored local download directory.
- `scripts/prepare_uci_online_retail_sample.py`: reproducible preparation script.

## Fields Used

- `InvoiceNo`: mapped to FinCore payment references such as `UCI-536365`.
- `InvoiceDate`: stored in payment metadata.
- `CustomerID`: retained as the dataset's anonymized/customer identifier, not a FinCore user identity.
- `Country`: stored in payment metadata.
- `Quantity` and `UnitPrice`: aggregated into integer minor-unit GBP payment amounts.
- `StockCode` and `Description`: retained only for a short line-item sample in metadata.

## Transformations

1. Positive line items are grouped by `InvoiceNo`.
2. Each grouped invoice becomes a captured merchant payment from the dataset customer wallet to the dataset merchant wallet.
3. Matching cancellation invoices are represented as refunds when the sample contains a selected payment for the same dataset `CustomerID`.
4. Missing `CustomerID`, non-positive quantities, and non-positive prices are excluded.
5. Values are converted to integer GBP minor units using decimal rounding.
6. Seeded withdrawal and reconciliation records are generated by FinCore services after payments/refunds are posted.

## Reproduction

```bash
python scripts/prepare_uci_online_retail_sample.py
cd apps/api
FINCORE_DATABASE_URL=sqlite:///./fincore-local.db alembic upgrade head
FINCORE_DATABASE_URL=sqlite:///./fincore-local.db python -m fincore.public_dataset_seed
```

Repeated seed execution is idempotent: existing `UCI-*` payments, refunds, webhook endpoint, and payout sample are not duplicated.

## Limitations And Ethics

The source is retail invoice data, not private banking, card, or bank-account data. FinCore maps it into wallet and merchant-payment workflows for engineering demonstration only. The dataset does not prove regulatory readiness, payment-network connectivity, fraud-detection accuracy, or production financial performance.
